# Bootstrapper

## Zweck
Der Bootstrapper ist die erste Komponente der Anwendung.

## Aufgaben
- Betriebssystem erkennen
- Hardware erkennen
- Verfügbare APIs erkennen
- Runtime-/BootConfig erzeugen
- Geeignetes UI auswählen
- Informationen an den Core übergeben

## Nicht Aufgabe
- Keine Geschäftslogik
- Keine Chat-Verarbeitung
- Keine KI-Ausführung
- Keine Datenbanksuche
- Keine Modulimplementierungen
